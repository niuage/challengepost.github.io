Rails exercise

Let's build a mini-Devpost in Ruby on Rails!
This exercise is straightforward and will allow you to demonstrate your knowledge of the Rails fundamentals.

The application

Business rules

Let's get down to business:

  • Users can create Competitions.
  • Competitions can have multiple prizes
  • Users can be free users or paid users
  • Free users can add a maximum of 3 prizes to each of their Competitions
  • Paid users can add as many prizes as they want to each of their Competitions

Suggested models

Feel free to name these models or attributes as you see fit.

Competition Prize User

Name (255 chars max)

Name (1000 chars max)

Dollar value (cannot be negative)

Full name

Free or paid

Application pages

User page

It should display:

  • the user name

  • a list of their Competitions (don't bother with pagination)

    For each Competition, display:

    • their name
    • their total prize value (i.e. the sum of the prize values).
    • all their prizes

    For each prize, display:

    • their name
    • their value
Keep performance in mind here ;)

Competition creation form

A single form allowing a signed in user to create a To-Do along with its prizes.

Do not bother with Javascript at this stage, you can make it so there are a fixed number of fields to add prizes.

To-Do update form

Same as above, but to edit an existing Competition. This page should only be accessible to the Competition owner.

ps: We're not asking you to make prizes deletable.

All the forms should display error messages for invalid inputs.

Adding comments to your code to explain your choices, or briefly mention alternatives you considered is encouraged.
