The To-Do application

Business rules

Let's get down to business.

  • Users can have multiple To-Dos
  • To-Dos can have multiple tasks
  • Users can be free users or paid users
  • Free users can add a maximum of 3 tasks to any given To-Do
  • Paid users can add as many tasks as they want to any given To-Do

Suggested models

Feel free to name these models or attributes as you see fit.

Todo Task User

Title (255 chars max)

Description (1000 chars max)

Done (completion state)

Full name

Free or paid

Application pages

User page

It should display:

  • the user name

  • a list of their To-Dos (don't bother with pagination)

    For each To-Do, display:

    • their title
    • their state. The state of a To-Do is derived from the state of its tasks. A To-Do is "done" only if all its tasks are done.
    • all their tasks

    For each task, display:

    • their description
    • their state (done/not done, ‚úî/‚úò, ... up to you)

Keep performances in mind here ;)

To-Do creation form

A simple form allowing a signed in user to create a To-Do, and add tasks to it.

Do not bother with Javascript at this stage, you can make it so there are a fixed number of text fields to add tasks.

To-Do update form

Same as above, but to edit an existing To-Do. This page should only be accessible to the To-Do owner.

ps: We're not asking you to make tasks deletable.

All the forms should display error messages for invalid inputs.
